Before we start, check in if you haven't done so. We'll match up those without teams in a bit.

Let's go over a few technical items to avoid confusion.

You should have 1 laptop for your team that is ready to go for writing C, C++, or Java.

Your code will be compiled on Linux, but it's just fine to work on Windows or Mac.
We'll only be using standard libraries anyway.

All submissions are subject to a secret time limit; take too long, and your process is killed.
Hint: Efficient code.

If you use C or C++, you must use `int main()` rather than `void main()` and NEVER use `system("pause")`.

Do not prompt for input. Anything your program prints is considered output and will be judged.

Read each problem carefully and pay attention to the given input bounds.

Finally, remember that competition programming problems are difficult. Persistence pays off here.

For returning CodePSU competitors, we've made a few changes since last fall.

We've switched to the submission system used in the international ACM ICPC competition.
Bye CodePSU Drupal module :(

In order to be more like the ACM's ICPC, all problems are weighted equally.

The submission system needs Java JDK installed on your computer to run. It's a big download, so do that now if you haven't already.

You'll also need to download the submission system from http://acm.psu.edu/sites/all/files/codepsu-submissions-teams.zip.
We'll go over this. Don't worry if you don't catch that long URL.
